# 01 - 产品需求文档（PRD）

> Project DOUYIN-FX ｜ v0.1（重写版 - 个人内部工具定位）
> 文档状态：草案 / 待自评

## 0. 关键定位（与上一版的差异）

上一版把项目定位为"多平台特效工程化平台"，与 1 人团队、内部使用、纯抖音场景严重不匹配，已归档到 `_archive-v0/`。

**当前定位（极小化重写）**：

> 一个**给我自己用**的小工具，**用 AI 模型快速生成抖音面部特效需要的素材**（贴图 / 序列帧 / LUT），并把素材**装配成可以直接拖进抖音 Effect Creator 的资产包**。
>
> **不**产出抖音工程文件本身，**不**替代 Effect Creator，**不**做多平台。
>
> Effect Creator 里手工搭最后那一步工程 + 提交审核，由我自己完成。

工具的真实价值 = 把"想一个特效 → 在 Effect Creator 里上架"的耗时从 **数天** 压到 **数小时**，瓶颈从"画素材"变成"想创意"。

## 1. 背景与动机

- 抖音 Effect Creator 是上架抖音特效的官方编辑器，个人创作者免费可用，**最终上架必须经过它**。
- Effect Creator 自带的素材生成能力很弱：贴图 / 序列帧 / LUT 都需要外部美术工具产出。
- 单人没有美术资源，但 AI 模型（Stable Diffusion / SDXL Turbo / ControlNet 等）可以快速产出风格化贴图、序列帧、LUT。
- 当前痛点：
  - 用 ComfyUI / Auto1111 跑 SD 出图后，要手动改尺寸、抠通道、对齐 UV 模板、命名归档 → 重复劳动占大头
  - 没有统一的"prompt → 抖音可用素材包"流水线，每次都从零拼
  - 出图参数 / seed / 模型版本散在多个工具里，不易复刻
- 目标：用 1 个工具把"调模型 + 后处理 + 素材打包"串成 1 条流水线，**人只负责想 prompt 和搭工程**。

## 2. 目标（Goals）

- **G1**：把"prompt → 抖音可用素材包"流程**端到端跑通**，单次出素材时长 ≤ 10 分钟（含模型推理）。
- **G2**：素材包内容明确：图片 / 序列帧 / LUT 三类；按 Effect Creator 期望的命名、尺寸、格式直接落盘。
- **G3**：每个素材保留 prompt / seed / model_id / 后处理步骤，下次想复刻或微调能找回来。
- **G4**：本地优先 + 远程兜底。本机能跑就本地（macOS Apple Silicon），跑不动调远程 OpenAI 兼容 endpoint。
- **G5**：v0.1 跑通 1 个示例特效全流程：**用工具产出素材 → 在 Effect Creator 里搭工程 → 提交抖音审核 → 上架**。

## 3. 非目标（Non-Goals，明确不做）

- ✗ **不做**抖音 Effect Creator 工程文件（.dfx / 工程 zip）的产出 / 解析 / 反向工程
- ✗ **不做**节点图编辑器、可视化连线、Tauri / Electron 桌面壳
- ✗ **不做**多平台支持（剪映 / TikTok / 视频号 / Lens Studio / Spark AR 全部砍掉）
- ✗ **不做**多人协作 / 云端同步 / 商业化 / 公开发布
- ✗ **不做**自研模型微调、训练、LoRA 训练（v0.1 只调用现成模型）
- ✗ **不做**直接对接抖音上架 API（即使有也不应该用，正常走 Effect Creator 提审）
- ✗ **不做**"安全合规中间件"工程化（NSFW classifier / prompt 黑名单 / 审计日志），只在文档层做"自我提醒清单"，由我自己负责合规

## 4. 用户与场景

### 4.1 用户

- **唯一用户：我自己**（项目作者 / 内部使用）。
- v0.1 不向他人分发；如果未来想给朋友用，再单独评估。

### 4.2 核心场景

工具围绕 3 类抖音常见特效素材构建：

| 场景 | 说明 | 工具产出 | 我手工做的部分 |
|---|---|---|---|
| **S1：面部贴图特效** | 赛博面具 / 妆容 / 纹身 等 | 1 张对齐 face_uv 模板的 PNG（带 alpha） | 在 EC 里挂到 face mesh，调透明度 / 触发器 |
| **S2：动态贴纸特效** | 火焰皇冠 / 翅膀 / 漂浮粒子 等 | 一组带 alpha 的序列帧 PNG（30-60 帧） | 在 EC 里挂到 head anchor，循环播放 / 触发 |
| **S3：色彩滤镜** | 港片调色 / 复古胶片 / 电影感 等 | 1 个 .cube LUT 文件 + 1 张参考效果图 | 在 EC 里加 LUT 滤镜组件 |

v0.1 必须跑通 **S1**，S2/S3 v0.2 跟进。

### 4.3 工作流（重要）

```
[我]                  [工具]                       [Effect Creator]      [抖音]
 想 prompt   ──→   调模型 + 后处理 + 打包    ──→     手工搭工程    ──→   提审 + 上架
              （工具的全部范围）                   （EC 官方流程）       （抖音平台）
```

工具只负责中间那一段。前后两段是人 + 平台。

## 5. v0.1 范围

### 5.1 v0.1 必做（MUST）

| ID | 能力 | 验收 |
|---|---|---|
| F1 | 本地 Web UI（Gradio 或 FastAPI + 简单 HTML），单机启动 | `python -m douyinfx` 启动后浏览器可访问 |

| F6 | 素材包打包：1 个 zip，里面放贴图 + 元数据 JSON（prompt/seed/model_id/created_at） | unzip 后结构正确 |
| F7 | 远程后端兜底：填一个 OpenAI 兼容 endpoint，本地跑不动时切远程 | 配置后能切，不改代码 |

- 多平台

## 6. 非功能性需求

| 维度 | 目标 | 备注 |
|---|---|---|
| 启动 | 浏览器打开 UI ≤ 5s（不含模型加载） | 模型首次加载 30-60s 可接受 |
| 出图（本地 mps） | SDXL Turbo 单图 ≤ 8s @ 1024×1024 | M1 Pro 16GB 基线 |
| 出图（远程） | 取决于 endpoint，工具不承诺 | — |
| 内存 | 空闲 ≤ 1GB；推理峰值 ≤ 12GB | M1 Pro 16GB 基线 |
| 平台 | macOS Apple Silicon 唯一支持平台 | Windows / Intel / Linux 不保证 |
| 安装 | `git clone + pip install -e .` 跑通 | 不打包安装包 |
| 日志 | 本地文件日志，纯文本，方便我自己看 | 不结构化、不上传 |

## 7. 系统边界与外部依赖

| 依赖 | 性质 | 风险 |
|---|---|---|
| 抖音 Effect Creator | 我手工使用，工具不调用 | 平台升级改资产规范 → 改工具规范化逻辑 |
| Stable Diffusion / SDXL Turbo 权重 | 从 HuggingFace / 国内镜像下载 | 国内 HF 不稳，需配镜像 |
| Diffusers / PyTorch / mps | Python 库 | mps 后端偶尔有 bug，准备 cpu 兜底 |
| rembg / onnxruntime | 抠图 | 稳定 |
| 远程 endpoint（用户自填） | OpenAI 兼容 | 网络 / 余额 / 协议变化 |

## 8. 合规自检清单（个人责任，非工程化）

> 这是给"我自己"看的清单，**不是工程化拦截器**。每次导出素材包前 UI 弹窗让我勾选。
> 因为只是内部工具且最终走抖音官方审核，不需要工程化合规中间件。
> 但**上架抖音时这些是抖音审核的硬门槛**，不能省。

每次导出素材包前必须自检：

- [ ] **不涉及真实政治人物 / 公众人物 / 名人脸部特征**（抖音对深度伪造红线）
- [ ] **不涉及色情、露点、性暗示**（抖音社区规范）
- [ ] **不涉及血腥、自残、暴力、恐怖元素**
- [ ] **不涉及他人肖像 / 商标 / 受版权保护 IP**（哆啦 A 梦 / 米老鼠 / 名牌 logo 等）
- [ ] **不涉及政治敏感、民族 / 宗教 / 性别歧视**
- [ ] **不涉及违法物品**（毒品 / 武器 / 赌博）
- [ ] **生成内容确认是 AI 生成**（按《互联网信息服务深度合成管理规定》《生成式人工智能服务管理暂行办法》要求，上架时按抖音模板填"AI 生成"标签；元数据 JSON 标记 `ai_generated: true`）

工具实现：UI 弹窗显示 7 项 checkbox，全部勾选才允许"导出 zip"按钮可点。**这只是物理提醒，不是过滤器**。

## 9. 数据与隐私

- 工具单机运行，**不收集任何数据**。
- 工具不调用摄像头（v0.1 不需要，因为工具不做实时预览，所有素材在 EC 里跑摄像头）。
- 远程 endpoint 我自己填的 endpoint，prompt 和参考图会上传 → **我自己对此负责**，工具只在 UI 上提示一次。
- 所有产出落盘到 `~/douyin-fx-projects/<日期>-<特效名>/`。

## 10. 验收基线（Definition of Done）

v0.1 完成的标志：

- [ ] 我自己用工具跑通场景 S1 全流程：输入 prompt → 出图 → 抠背景 → 导出 zip → 在 EC 里搭工程 → 在 EC 模拟器里看到贴图随表情移动
- [ ] 至少把 1 个特效提交到抖音审核（不要求过审，验证流程能走完）
- [ ] 文档（本 PRD + 技术方案 + 工作流手册 + 合规自检 + README）齐全

## 11. 中国大陆合规要点（仅文档层提醒）

> 工具本身不直接触发监管（单机内部不分发），但**上架到抖音的特效**会被抖音审核 + 平台合规义务覆盖。

- 《互联网信息服务深度合成管理规定》（2023）：深度合成生成的内容须显著标识；面部特效若属"深度合成"范畴，抖音侧已有标识机制，提审时按抖音指引填即可。
- 《生成式人工智能服务管理暂行办法》（2023）：**面向公众提供** AI 生成服务才需要备案；本工具内部使用、不向公众提供，不在适用范围；但**上架到抖音的特效**由抖音承担其平台侧义务，与我无关。
- 《民法典》肖像权 / 著作权法：不能拿别人的脸 / 别人的画做素材。
- 《网络安全法》《个人信息保护法》：本工具不处理个人信息，不适用。

**结论**：v0.1 阶段，我个人在 §8 的自检清单里把好关 + 抖音官方审核兜底，已经足够。不需要工程化的 NSFW / 黑名单 / 审计日志。

| ID | 风险 | 影响 | 缓解 |
|---|---|---|---|
| R1 | Effect Creator 资产规范我没吃透，工具产出的素材 EC 不认 | 高 | 写代码前先用 EC 跑一个完整 demo 特效，把"EC 真实接受什么样的素材"摸清，再回头写工具 |
| R2 | SDXL Turbo 在 mps 上不稳定 / 速度不达预期 | 中 | 准备远程 endpoint 兜底；或退回到 SD 1.5 + LCM-LoRA |
| R3 | 国内 HuggingFace 拉权重失败 | 高 | 用 ModelScope 或 hf-mirror.com 镜像；提供"手动放权重到 ~/.cache/huggingface/hub"的说明 |
| R4 | 我自己投入的时间被低估 | 中 | v0.1 砍到只做 S1，2 周内出第一版；超 2 周就停下重新评估 |

| R6 | 上架后被抖音判定为 AI 生成需打标 | 低 | 提审时主动按抖音指引标注 AI 生成，按规走 |
