# 06 - 模型集成与后端路由

> Project DOUYIN-FX ｜ v0.1
> 文档状态：草案

## 1. 设计目标

- **能力 vs 实现解耦**：节点声明 capability（如 `text2img`），由 Model Hub 路由到具体 Provider。
- **多后端无感切换**：同一 capability 可由 CPU ONNX / Apple MPS / NVIDIA CUDA / 远程 HTTP 实现，用户切换不改工程。
- **License 合规**：模型权重不打包、不二次分发；用户加载前显式确认 license。
- **资源可控**：单进程不允许同时常驻多个重型模型；按 LRU 卸载。

## 2. 核心抽象

### 2.1 Capability

| Capability | 输入 | 输出 | 内置实现 |
|---|---|---|---|
| `facemesh` | RGB 图 | landmarks + pose | MediaPipe FaceMesh |
| `matting` | RGB 图 | mask | MediaPipe Selfie / SAM |
| `depth` | RGB 图 | 深度图 | MiDaS |
| `text2img` | prompt 等 | RGBA 图 | SD 1.5 / SDXL Turbo |
| `img2img` | RGBA 图 + prompt | RGBA 图 | SD 系列 |
| `inpaint` | 图 + mask + prompt | RGBA 图 | SD inpaint（v0.1 选做） |
| `style_transfer` | RGBA 图 + style_id | RGBA 图 | fast-style-transfer |
| `nsfw_classify`（内部） | RGBA 图 | score | open-nsfw 同等 |
| `prompt_classify`（内部） | text | label | 关键词 + 轻量分类器 |

### 2.2 ModelProvider

```python
class ModelProvider(Protocol):
    capability: str
    backend: str               # "cpu_onnx" / "mps" / "cuda" / "remote_http"
    model_id: str
    version: str
    license: License

    def warm_up(self) -> None: ...
    def infer(self, payload: dict) -> dict: ...
    def release(self) -> None: ...
    def signature(self) -> str:
        """稳定签名（含 model_id + backend + version + 关键参数），用于缓存键。"""
```

### 2.3 ModelHub

- **注册中心**：所有 Provider 启动时注册。
- **路由器**：节点请求 capability → 按用户偏好 + 硬件可用性挑 Provider。
- **生命周期**：负责 warm_up / release，按 LRU 卸载常驻模型。

## 3. 后端路由

### 3.1 默认优先级

```
cuda > mps > cpu_onnx > remote_http
```

可由用户在设置中按 capability 单独覆写。

### 3.2 硬件探测

- 启动时探测：CUDA + 显存、Apple Silicon + 统一内存、CPU + 内存、ONNX Runtime EP 列表。
- 探测结果落盘 `~/.douyinfx/hardware_profile.json`，重启复用；用户可手动重新探测。

### 3.3 远程后端

- 用户自填 endpoint，工具不内置任何默认上传地址。
- 协议（详见 §6）：OpenAI 兼容 / ComfyUI / 自定义 JSON-RPC。
- 上传前 UI 显式提示"以下数据将上传到 {endpoint}"，需用户确认；可勾选"本工程内不再提示"。
- 远程后端失败 → 自动降级到本地后端，**不静默切换**，UI 弹通知。

## 5. License 管理

### 5.1 License 类型

```python
class License(BaseModel):
    spdx: str                       # "Apache-2.0" / "MIT" / "CreativeML-OpenRAIL-M" ...
    full_text_path: Path
    requires_user_consent: bool
    commercial_use: bool
    attribution_required: bool
    attribution_string: str | None
```

### 5.2 用户确认流程

- 首次加载某 model_id：
  1. 弹窗显示 license 全文 + 商用 / 署名要点
  2. 用户必须勾选"我已阅读并接受"
  3. 写入 `~/.douyinfx/model_consent.json`
- 同 model_id 同 version 不再弹窗；version 变更需重新确认。
- consent 文件**仅本地**，不上传。

### 5.3 商业使用提示

- 模型 `commercial_use=False` 时，UI 在节点上加"非商用"标记。
- 导出报告中列出本工程涉及的非商用模型，并提示用户。
- 工具不阻止导出；商用决策由用户负责，但工具尽到提示义务。

## 6. 远程后端协议

### 6.1 OpenAI 兼容（默认）

```
POST {endpoint}/v1/images/generations
Authorization: Bearer {api_key}
{ "model": "...", "prompt": "...", "n": 1, "size": "1024x1024", "response_format": "b64_json" }
```

### 6.2 ComfyUI

- 用户填 ComfyUI 服务地址 + workflow 模板路径
- 工具按模板填 prompt / seed 等占位
- WebSocket 拉进度

### 6.3 自定义 JSON-RPC

- 用户自填 schema 配置文件，描述请求 / 响应字段映射
- 工具按 schema 翻译，不感知具体协议

### 6.4 数据上传规则

- **仅上传节点输入**（prompt / 上游图像 / mask），不上传整工程。
- 上传前 UI 显示原始 payload 摘要（prompt 全文 + 资产哈希），用户确认后发出。
- 用户可在设置里关闭某 endpoint 的所有出站请求。

## 7. 缓存策略

### 7.1 推理结果缓存

- 缓存键：`hash(node_type + ModelProvider.signature() + canonical_json(input))`
- 位置：`~/.douyinfx/cache/<capability>/<model_id>/<key>.{ext}`
- 上限：用户可配置（默认 10GB）；超限按 LRU 淘汰。

### 7.2 模型权重缓存

- 位置：`~/.douyinfx/models/<model_id>/<version>/`
- 校验：sha256
- 更新：用户主动"检查更新"，工具不静默升级

## 9. 安全与合规

- **不内置**任何换脸 / 人脸 ID 识别 / 活体破解模型。
- **不内置**针对特定真人 / 明星 / 政治人物的人脸 LoRA / Embedding。
- v0.1 锁定"只能加载内置清单中的模型"；自定义 LoRA / 自定义 SD 进 v1（详见 §10）。
- 远程 endpoint 返回的违规结果走本地 NSFW 过滤兜底。

## 10. 自定义模型加载（v1）

- 用户加载 LoRA / 自定义 SD 模型前：
  - 工具计算文件哈希，与本地黑名单（可热更新）比对
  - 显示"加载非官方模型，使用风险自负"弹窗
- 加载行为写审计日志。
- 不允许用户自定义 `nsfw_classify` / `prompt_classify` 模型（防止用户禁用安全过滤）。

## 11. 故障与回退

| 故障 | 行为 |
|---|---|
| GPU OOM | 自动降级到 CPU 后端 + 通知 |
| 远程 endpoint 不可达 | 重试 3 次后降级到本地后端 + 通知 |
| 模型权重哈希校验失败 | 拒绝加载 + 引导重新下载 |
| ONNX Runtime EP 加载失败 | 降级到下一优先级 EP |
| 推理结果异常（NaN / 全黑） | 节点标红，丢弃缓存，下次重算 |
