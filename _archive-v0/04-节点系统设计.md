# 04 - 节点系统设计

> Project DOUYIN-FX ｜ v0.1
> 文档状态：草案

## 1. 设计目标

- **强类型端口**：编辑期错连线即拒，运行期再校验一次。
- **声明式参数**：节点参数 schema 化，UI 自动渲染表单，避免手写 UI。
- **执行模型可预测**：节点是纯函数（参数 + 输入 → 输出 + 副作用清单），副作用全部声明。
- **可缓存可烘焙**：每个节点产出能落盘，下游引用哈希；重型节点支持显式烘焙。
- **可扩展**：新增节点 = 加一个文件 + 注册元数据，不改框架。

## 2. 节点元模型

```python
class NodeSpec:
    type: str                    # "input.camera" / "track.facemesh" / ...
    category: str                # "input" / "track" / "model" / "image" / "render" / "logic" / "output"
    label: str                   # UI 显示名
    inputs: list[PortSpec]       # 输入端口
    outputs: list[PortSpec]      # 输出端口
    params: list[ParamSpec]      # 节点参数（不通过端口传入）
    cost: NodeCost               # 性能成本档位（cheap/medium/heavy/bake_only）
    side_effects: list[str]      # "camera" / "filesystem" / "network" / "model_inference"
    platform_capability: dict    # 各平台支持矩阵，供 Exporter 决定降级策略
```

## 3. 端口类型系统

| 端口类型 | 数据形态 | 备注 |
|---|---|---|
| `Image` | RGBA 图像（Tensor / numpy）+ 时间戳 | 主要数据流 |
| `Mask` | 单通道 float32 [0,1] | 抠图 / 分割结果 |
| `Mesh` | 顶点 + UV + 法线（FaceMesh / 自定义） | 跟踪节点输出 |
| `Landmarks` | (N, 3) float32 | 关键点（人脸 / 手 / 身体） |
| `Pose` | 6DoF 矩阵 | 头部 / 物体姿态 |
| `Float` / `Vec2` / `Vec3` / `Vec4` | 标量 / 向量 | 参数信号 |
| `Asset` | 落盘资产引用（哈希 + 路径） | 模型生成产物 |
| `Trigger` | 事件流（boolean + 时间戳） | 张嘴 / 眨眼 / 定时 |
| `Sequence<T>` | 序列帧 / 时间轴 | 烘焙产物 |

**连线规则**：

- 同类型直接连。
- `Sequence<Image>` 可隐式降级为 `Image`（取当前帧）。
- 其他类型转换需用显式转换节点（如 `mask.to_image`）。

## 4. 节点目录（v0.1 范围）

### 4.1 输入类（input）

| 节点 | 输入 | 输出 | 参数 | 备注 |
|---|---|---|---|---|
| `input.camera` | — | `Image` | device_id, resolution, fps | 摄像头采集 |
| `input.video_file` | — | `Image` | path, loop | 测试用 |
| `input.image_file` | — | `Image` | path | 静态图 |

### 4.2 跟踪类（track）

| 节点 | 输入 | 输出 | 参数 | 备注 |
|---|---|---|---|---|
| `track.facemesh` | `Image` | `Mesh`, `Landmarks`, `Pose` | model_complexity, max_faces | MediaPipe FaceMesh |
| `track.head_pose` | `Landmarks` | `Pose` | smoothing | 从 landmarks 解算 |
| `track.matting` | `Image` | `Mask` | model（mediapipe / sam）, threshold | 人像分割 |
| `track.depth` | `Image` | `Image`（深度图） | model（midas） | 单目深度 |

### 4.3 模型生成类（model）

> **重型节点**，默认走烘焙模式；编辑期使用上次缓存。

| 节点 | 输入 | 输出 | 参数 | 备注 |
|---|---|---|---|---|
| `model.text2img` | — | `Image`, `Asset` | prompt, negative_prompt, seed, steps, cfg, model_id, lora | Stable Diffusion |
| `model.img2img` | `Image` | `Image`, `Asset` | prompt, strength, seed, steps, cfg | 图生图 |
| `model.style_transfer` | `Image` | `Image` | style_id, strength | 快速风格化 |
| `model.inpaint` | `Image`, `Mask` | `Image`, `Asset` | prompt, seed | 局部重绘（v0.1 选做） |

所有 `model.*` 节点：

- prompt 必走 §7 安全过滤。
- 出图必走 NSFW 二次过滤。
- 产物自动落盘到 `assets/`，元数据写 sidecar JSON。

### 4.4 图像处理类（image）

| 节点 | 输入 | 输出 | 参数 | 备注 |
|---|---|---|---|---|
| `image.compose` | `Image` × N, `Mask`（可选） | `Image` | blend_mode, opacity | alpha 合成 |
| `image.lut` | `Image` | `Image` | lut_path | 颜色查表 |
| `image.adjust` | `Image` | `Image` | brightness, contrast, saturation, hue | 基础调色 |
| `image.uv_apply` | `Mesh`, `Image` | `Image` | uv_layout | 把贴图贴到 mesh UV |
| `image.sequence` | — | `Sequence<Image>` | folder_path, fps | 序列帧 |

### 4.5 渲染类（render）

| 节点 | 输入 | 输出 | 参数 | 备注 |
|---|---|---|---|---|
| `render.preview` | `Image` | — | overlay_fps, overlay_inference_ms | 预览窗输出 |
| `render.recorder` | `Image` | — | path, duration | 录制对照 |
| `render.export_target` | `Image` / `Mesh` / ... | — | export_target | 标记为某平台导出端点 |

### 4.6 逻辑 / 时间轴类（logic）

| 节点 | 输入 | 输出 | 参数 | 备注 |
|---|---|---|---|---|
| `logic.trigger.mouth_open` | `Landmarks` | `Trigger` | threshold | 张嘴触发 |
| `logic.trigger.blink` | `Landmarks` | `Trigger` | threshold | 眨眼触发 |
| `logic.timer` | — | `Trigger` | interval_s, mode | 定时器 |
| `logic.state_machine` | `Trigger` × N | `Float` | states, transitions | 简单状态机 |
| `logic.lerp` | `Float`, `Trigger` | `Float` | from, to, duration | 触发驱动的插值 |

### 4.7 烘焙类（bake，元节点）

| 节点 | 输入 | 输出 | 参数 | 备注 |
|---|---|---|---|---|
| `bake.image_sequence` | 任意上游子图 | `Sequence<Image>` | frames, fps | 把上游烘焙为序列帧 |
| `bake.image` | 任意上游子图 | `Image`, `Asset` | resolution | 烘焙单帧 |

烘焙节点是 Exporter 能力降级的兜底（详见 `05-多平台导出器规范.md`）。

## 5. 节点参数 Schema

参数声明示例（pydantic）：

```python
class Text2ImgParams(BaseModel):
    prompt: str = Field(..., max_length=2000, description="正向描述")
    negative_prompt: str = Field("", max_length=2000)
    seed: int = Field(-1, ge=-1, le=2**31 - 1, description="-1 表示随机")
    steps: int = Field(20, ge=1, le=100)
    cfg: float = Field(7.0, ge=1.0, le=20.0)
    model_id: str = Field("sd15", description="model_id from model hub")
    lora: list[LoraRef] = Field(default_factory=list, max_items=4)
```

UI 自动按 schema 渲染：`int → 滑块`，`float → 滑块`，`str(max_length<=200) → 单行`，`str(>200) → 多行`，`enum → 下拉` 等。

## 6. 执行模型

### 6.1 三种执行模式

| 模式 | 触发 | 行为 |
|---|---|---|
| **预览（preview）** | UI 实时 | 每帧执行 DAG，重型节点取缓存，不实时跑 |
| **烘焙（bake）** | 用户点烘焙按钮 / 节点标记 always_bake | 完整跑一次重型节点，产物落盘 |
| **导出（export）** | 用户点导出 | 完整跑一次 DAG，所有需要的资产固化，交给 Exporter |

### 6.2 缓存键

```
cache_key = sha256(
    node_type ||
    node_version ||
    canonical_json(params) ||
    upstream_output_hashes ||
    model_provider_signature
)
```

- `model_provider_signature` = 模型 ID + 后端 + 后端版本（保证同 prompt 在不同后端不串缓存）。
- 缓存命中策略：**严格一致才命中**，宁可重算不要错命中。

### 6.3 增量计算

- 节点参数变更 → 标脏自身 + 全部下游传递闭包。
- 上游输出哈希未变 → 当前节点保留缓存。
- 编辑器拖拽 / 撤销 重做 走相同标脏路径，UI 与引擎共用一份脏集合。

### 6.4 错误处理

- 节点执行抛异常 → 捕获、UI 节点标红、错误信息附在节点上。
- 下游节点收到 "上游错误" 信号 → 跳过执行，传播错误状态。
- 导出器在 export 模式遇错 → 拒绝出包，输出包含完整错误链的导出报告。

## 7. 安全合规钩子

- **prompt 入口拦截**：`model.text2img` / `model.img2img` / `model.inpaint` 在执行前调用 `core/safety/prompt_filter.py`，命中黑名单 → 拒绝执行 + 节点标红 + 写审计日志。
- **生成结果过滤**：所有 `model.*` 输出图像在写入资产仓前过 NSFW classifier，命中 → 丢弃 + 写审计日志。
- **导出前扫描**：Exporter 在打包前再过一次工程内所有 `Image` / `Sequence<Image>` 资产，命中 → 拒绝出包。

详见 `07-安全合规与红线清单.md`。

## 8. 节点版本与迁移

- 每个节点带 `version` 字段（独立于 FX-IR 主版本）。
- 节点参数变更走"加字段 + 默认值"路径，老工程加载时由 `core/fxir/migrate/<node_type>.py` 填默认值。
- 节点废弃 → 标 `deprecated`，UI 仍可加载老工程但创建新工程时不出现在面板。

## 9. 节点开发规范（贡献者视角）

- 新节点放在 `py/nodes/<category>/<type>.py`。
- 必须实现：
  - `class XxxNode(Node):` 继承基类
  - `class XxxParams(BaseModel):` 参数 schema
  - `SPEC: NodeSpec = ...` 元数据
- 必须写：单测（参数边界 / 缓存键稳定性 / 错误传播）。
- 副作用必须显式声明在 `SPEC.side_effects`，未声明的副作用引擎会拒绝执行。
- UI 自动从 `NodeSpec` 渲染，不允许节点自带 UI 组件（v0.1 锁定，v2 再开放自定义 UI）。

## 10. 平台能力矩阵（节点 × 平台）

> 用于 Exporter 决定降级策略；详见 `05-多平台导出器规范.md` §3。

| 节点 | DYFX | 剪映滤镜包 | TikTok EH (v1) |
|---|---|---|---|
| `input.camera` | ✅（运行时） | ❌（需烘焙） | ✅ |
| `track.facemesh` | ✅ | ❌（需烘焙） | ✅ |
| `track.matting` | ✅ | ⚠️ 仅静态 | ✅ |
| `model.text2img` | ❌（需烘焙） | ❌（需烘焙） | ❌（需烘焙） |
| `model.style_transfer` | ⚠️ 烘焙优先 | ⚠️ LUT 近似 | ⚠️ 烘焙优先 |
| `image.uv_apply` | ✅ | ❌ | ✅ |
| `image.lut` | ✅ | ✅ | ✅ |
| `logic.trigger.mouth_open` | ✅ | ❌ | ✅ |

- ✅ = 平台原生支持
- ⚠️ = 部分支持 / 需近似实现
- ❌ = 不支持，需上游烘焙为静态资产
