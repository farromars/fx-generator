# 07 - 安全合规与红线清单

> Project DOUYIN-FX ｜ v0.1
> 文档状态：草案
> 本文档是 PRD §11 的可执行版本，所有红线条目都对应到代码层的拦截点。

## 1. 总原则

- **红线即拦截**：命中红线 → 直接拒绝执行 / 加载 / 导出，不接受"用户自负"绕过。
- **三道关卡**：prompt 入口 → 生成结果 → 导出前；每道独立，互为兜底。
- **审计可追溯**：所有命中事件写入本地审计日志，结构化 JSON，包含足够上下文（不含用户隐私）。
- **本地审计、不上报**：审计日志仅本地，工具不主动上报；用户可主动导出协助平台调查。
- **黑名单可热更新**：词表 / 分类器 / 模型黑名单走配置文件 + 远程拉取（用户授权后），不需要发版。

## 2. 红线清单

### 2.1 人脸相关红线

| 编号 | 红线 | 拦截点 |
|---|---|---|
| F-01 | 不提供"指定真人换脸"能力 | 节点层不存在 face_swap 节点；自定义模型加载前过名称黑名单 |
| F-02 | 不提供"明星 / 政治人物 / 公众人物"识别 / 生成 / 换脸 | prompt 黑名单 + LoRA 黑名单 + 生成结果二次审查 |
| F-03 | 不生成可用于身份证件 / 人脸验证攻击的素材（活体破解、3D 面具贴图） | prompt 关键词黑名单 + 节点级拦截（禁止 inpaint 在身份证模板上工作） |
| F-04 | 不内置 face recognition / face ID 模型 | 模型清单层硬编码，不接受自定义注入 |

### 2.2 内容红线

| 编号 | 红线 | 拦截点 |
|---|---|---|
| C-01 | 拒绝色情、性暗示、未成年人不适宜内容 | prompt 黑名单 + NSFW classifier（生成后 + 导出前） |
| C-02 | 拒绝血腥、暴力、自残、自杀相关内容 | prompt 黑名单 + 内容分类器 |
| C-03 | 拒绝政治敏感、民族 / 宗教 / 性别歧视、仇恨言论 | prompt 关键词黑名单 |
| C-04 | 拒绝恐怖主义、极端组织相关内容 | prompt 关键词黑名单 |
| C-05 | 拒绝违法物品（毒品、武器制造）相关内容 | prompt 关键词黑名单 |

### 2.3 版权红线

| 编号 | 红线 | 拦截点 |
|---|---|---|
| R-01 | 不内置受版权保护的 IP 素材（动漫角色 / 商标 / 影视截图） | 内置资产清单审查 |
| R-02 | 用户上传素材时需勾选"已确认版权归属" | UI 弹窗 + 写 consent 日志 |
| R-03 | 模型 license 显式展示，加载前需用户确认 | 见 `06-模型集成与后端路由.md` §5 |

### 2.4 数据合规

| 编号 | 红线 | 拦截点 |
|---|---|---|
| D-01 | 摄像头画面**仅本地**处理，不外传 | 本地通路硬编码；远程节点禁止订阅 camera 输出（除非用户在 UI 上明确放进节点图） |
| D-02 | 远程推理需用户每次确认 | UI 弹窗 + payload 摘要 |
| D-03 | 工具不收集行为埋点（v0.1） | 代码层无埋点 SDK 引入 |
| D-04 | 崩溃日志默认本地，用户主动提交才上传 | 上传前预览脱敏 |

### 2.5 平台合规

| 编号 | 红线 | 拦截点 |
|---|---|---|
| P-01 | 不绕过任何平台官方审核 | Exporter 只产工程，不替代平台 IDE 上架流程 |
| P-02 | 导出工程必须标注"AI 生成" | `ExportOptions.ai_generated_label` 强制 true（v0.1） |
| P-03 | 不提供批量刷量导出 API | v0.1 不公开 CLI 批处理 |

## 3. 拦截点实现

### 3.1 Prompt 入口拦截

文件：`py/core/safety/prompt_filter.py`

```python
class PromptFilter:
    def check(self, prompt: str, negative_prompt: str = "") -> FilterResult:
        """
        返回:
          FilterResult(action="allow")
          FilterResult(action="reject", category=..., matched=..., reason=...)
        """
```

- **关键词黑名单**：分类别（人脸/内容/版权），支持中英文多语言、同义词、模糊匹配（编辑距离 ≤ 2）。
- **轻量分类器**：兜底，覆盖黑名单遗漏的语义变体。
- 调用点：所有 `model.text2img / img2img / inpaint` 节点执行前。

### 3.2 生成结果过滤

文件：`py/core/safety/image_filter.py`

```python
class ImageFilter:
    def check(self, image: Image) -> FilterResult: ...
```

- **NSFW classifier**：score 阈值（默认 0.6，可在配置中收紧）。
- **可选**：人脸出现 + 与黑名单人物相似度比对（v1，v0.1 不做相似度模型，避免引入识别能力）。
- 调用点：`model.*` 节点产出资产写入资产仓前。

### 3.3 导出前复扫

文件：`py/exporters/_safety_scan.py`

- 扫描工程内所有 `Image` / `Sequence<Image>` 资产，过 NSFW classifier。
- 命中即拒绝出包，记录"哪些资产命中"到导出报告（不展示资产，仅展示哈希 + 节点位置）。
- 调用点：所有 Exporter 在 `validate()` 与 `export()` 前置环节。

### 3.4 自定义模型加载审查（v1）

文件：`py/core/safety/model_admit.py`

- 文件名 / 元数据黑名单匹配（"face_swap" / 明星名 / 政治人物名 / 公众人物名）。
- 文件哈希黑名单（远程可热更新，用户授权后拉取）。
- 命中即拒绝加载 + 写审计日志。

## 4. 黑名单管理

### 4.1 文件位置

```
~/.douyinfx/safety/
├── prompt_blacklist.toml            # 用户可见，仅限本地
├── model_blacklist.toml             # 用户可见，仅限本地
├── prompt_classifier.onnx           # 二级分类器
├── nsfw_classifier.onnx
└── audit.log                        # 审计日志
```

### 4.2 词表结构

```toml
schema_version = 1
[categories.face]
en = ["face swap", "deepfake", "face replace", ...]
zh = ["换脸", "明星脸", "深度伪造", ...]
celebrity_names = [...]              # 单独一组，支持模糊匹配
political_figures = [...]            # 单独一组

[categories.content_nsfw]
en = [...]
zh = [...]

[categories.content_violence]
...
```

### 4.3 热更新

- 工具内可"检查黑名单更新"，从用户在设置里配置的 URL 拉取（默认未配置）。
- 拉取前显示新增条目摘要，用户确认后才生效。
- 不主动联网更新，避免被动行为。

## 5. 审计日志

### 5.1 日志格式

```json
{
  "ts": "2026-06-10T19:53:00+08:00",
  "level": "warn",
  "event": "safety.prompt_rejected",
  "category": "face.celebrity",
  "matched": ["xxxxx"],
  "node_id": "n_8a3f...",
  "node_type": "model.text2img",
  "project_id": "proj_2f...",
  "tool_version": "0.1.0",
  "reason": "celebrity_name_blacklist"
}
```

- **不**写入：完整 prompt 原文（仅写命中关键词）、用户名、设备 ID、文件路径。
- **写入**：节点 ID、工程 ID（哈希）、命中类别、命中关键词、时间戳、工具版本。

### 5.2 日志生命周期

- 默认保留 90 天，超期滚动删除（用户可配置 0~365 天）。
- 用户可一键导出（脱敏后）配合平台调查。
- 工具自身不主动上报。

### 5.3 事件类型

| event | 触发 |
|---|---|
| `safety.prompt_rejected` | prompt 入口命中黑名单 |
| `safety.prompt_classifier_flagged` | 二级分类器命中 |
| `safety.image_nsfw_rejected` | 生成图过 NSFW |
| `safety.export_blocked` | 导出前复扫命中 |
| `safety.model_admit_rejected` | 自定义模型加载被拒（v1） |
| `safety.consent_recorded` | 用户接受 license / 版权确认 |
| `safety.remote_upload_confirmed` | 用户确认远程推理上传 |

## 6. 平台规范对接

- 导出工程内嵌 `ai_generated: true` 与生成器列表（含模型 ID）。
- 平台未来要求平台原生标注协议时，按平台规范同步（在 Exporter 的 `mappings/<platform_version>.py` 实现）。
- 平台社区规范（图像 / 视频内容、广告 / 营销、未成年人保护）作为"软约束"在导出报告中以"建议"形式展示，不强制阻塞。

## 7. 测试与验收

### 7.1 红线测试集

- 维护 `tests/safety/redline_cases.yaml`，覆盖每条红线 ≥ 3 个 case。
- CI 每次合并跑全量红线测试，命中率必须 100%。
- 测试集**不**包含真实违规素材，仅含触发关键词与合成测试图。

### 7.2 灰度 case

- 维护 `tests/safety/grayscale_cases.yaml`，覆盖"语义变体 / 同义替换 / 多语言混写"。
- 目标：命中率 ≥ 95%；定期评审漏网案例并迭代词表。

### 7.3 误杀监控

- 内测期收集"用户反馈被误杀"的 case，定期评审词表。
- 误杀率（合规 prompt 被拦截）目标 ≤ 1%。

## 8. 变更管理

- 红线条目（§2）变更必须走"产品 + 法务 + 平台关系"三方评审。
- 词表 / 分类器变更可由安全团队直接发布。
- 变更记录写入 `CHANGELOG.safety.md`，独立于工具主 changelog。

## 9. 不做清单（明确）

- ✗ 不在工具内置任何"绕过审核"开关 / 调试模式。
- ✗ 不接受"我是开发者 / 我是研究人员"为由解锁红线能力。
- ✗ 不提供"关闭安全过滤"的 UI 入口（v0.1 锁定）。
- ✗ 不接入任何会回传用户原始素材的远端审核服务（保持数据本地原则）。
