# 05 - 多平台导出器规范

> Project DOUYIN-FX ｜ v0.1
> 文档状态：草案

## 1. 设计目标

- 同一份 FX-IR 工程可被多个 Exporter 翻译为各平台原生工程。
- Exporter 之间互不感知，新增平台只加文件不改框架。
- 不可识别的节点走**显式降级**（烘焙为静态资产），降级行为在导出报告中如实呈现。
- 不绕过任何平台官方审核流程；导出器只产出"能在官方编辑器打开"的工程，最终打包审核走平台 IDE。

## 2. Exporter 接口

```python
class Exporter(Protocol):
    platform: str                      # "douyin_effect_creator" / ...
    target_versions: list[str]         # 支持的平台 IDE 版本
    schema_version: str                # 自身导出器协议版本

    def capabilities(self) -> ExportCapabilities:
        """声明该平台支持的节点类型 / 资产格式 / 性能预算。"""

    def validate(self, fx_ir: FXIR, options: ExportOptions) -> ValidationReport:
        """干跑：仅校验，不落盘。给出 errors / warnings / 降级清单。"""

    def export(self, fx_ir: FXIR, options: ExportOptions) -> ExportResult:
        """正式导出：产出工程目录 + 导出报告。"""
```

```python
class ExportCapabilities:
    supported_node_types: set[str]
    asset_format: dict[str, list[str]]      # {"image": ["png", "jpg"], "video": ["mp4"]}
    asset_max_resolution: tuple[int, int]
    asset_max_filesize_mb: int
    project_max_filesize_mb: int
    perf_budget_per_frame_ms: int
    runtime_limits: dict                    # 平台运行时上限（节点数 / 触发器数 / 状态机层级）

class ExportOptions:
    target_version: str
    output_dir: Path
    overrides: dict                         # FX-IR.export_targets 中该平台的覆写
    ai_generated_label: bool = True          # 强制标注 AI 生成
    bake_overrides: dict                    # 哪些子图强制烘焙

class ExportResult:
    project_dir: Path
    report_path: Path                       # HTML 报告
    warnings: list[Warning]
    bakery_assets: list[Path]               # 本次烘焙产生的资产
```

## 3. 能力降级策略

降级原则：**安全 > 一致性 > 性能 > 文件体积**。

| 情况 | 默认策略 | 用户可覆写 |
|---|---|---|
| 节点类型平台不支持 | 上游子图烘焙为序列帧 | ❌（必须降级） |
| 资产分辨率超平台上限 | 等比缩放 + 警告 | ✅（用户可强制原始尺寸但导出会失败） |
| 单资产文件大小超限 | 重压缩（PNG → JPG q85 / 序列帧抽帧） | ✅ |
| 工程总包超限 | 报告中列 Top N 大资产 + 拒绝出包 | ❌ |
| 性能预算超标 | 报告中给出"建议改用烘焙节点"提示 | ⚠️ 不静默修改用户工程 |
| 触发器数超平台上限 | 报告中列触发器、拒绝出包 | ❌ |
| 节点参数超平台范围 | clamp 到合法范围 + 警告 | ✅ |

降级日志写入 `ExportResult.warnings`，HTML 报告中按"严重 / 警告 / 信息"三级渲染。

## 4. 抖音 Effect Creator 导出器（DYFX）

### 4.1 范围

- 平台：抖音 Effect Creator（中国大陆抖音）。
- 目标版本：v0.1 锁定一个稳定版本号（M0 调研期确定，后续按版本维护）。
- 工程结构：以平台官方开发者文档 / 内部资源为准；本文档不复刻平台私有结构，仅给"导出器视角"的契约。

### 4.2 输入 / 输出

| 类型 | 内容 |
|---|---|
| 输入 | FX-IR + ExportOptions |
| 输出 | DYFX 工程目录（可被官方编辑器直接打开）+ HTML 导出报告 |

### 4.3 节点映射约定

> 仅给原则，具体映射表随平台版本演进，维护在 `exporters/douyin_effect_creator/mappings/<platform_version>.py`。

| FX-IR 节点 | DYFX 表达 |
|---|---|
| `input.camera` | 平台默认相机输入（不需要导出） |
| `track.facemesh` | 平台原生 face tracker，绑定 anchor |
| `track.matting` | 平台原生 segmentation |
| `image.uv_apply` | UV mesh 贴图组件 |
| `image.lut` | LUT 滤镜组件 |
| `image.compose` | 合成图层（按 blend_mode 映射） |
| `model.*` | 烘焙为序列帧 → 序列帧组件 |
| `logic.trigger.mouth_open` | 平台表情触发器 |
| `bake.image_sequence` | 序列帧资产 |
| `render.export_target` | 工程出口 |

### 4.4 资产规范

- 图像：PNG（带 alpha）/ JPG（不带 alpha 时优先）。
- 序列帧：PNG sequence，按帧号命名 `frame_0001.png`。
- 总包大小：以平台公开上限为准（M0 调研确定）。
- 分辨率：以平台公开上限为准。
- 文件名：仅 ASCII，避免空格 / 中文（部分平台 IDE 对路径敏感）。

### 4.5 元数据要求

- 导出工程内嵌 manifest（`douyinfx_meta.json`），记录：
  - 工具版本、FX-IR 版本、导出器版本、目标平台版本
  - 是否 AI 生成（强制 `true`）
  - 资产清单（哈希 + 来源）
- **不**写入用户私有信息（电脑名 / 路径 / 设备 ID）。

### 4.6 验证清单（导出前）

- 工程内是否含 `render.export_target` 节点（无则拒绝）。
- 是否命中 §6 安全合规复扫。
- 资产分辨率 / 大小 / 总包 / 触发器数 是否在限内。
- 平台不支持的节点是否已被降级或上游烘焙。

## 5. 剪映 / CapCut 滤镜包导出器

### 5.1 范围

- 平台：剪映（中国大陆） / CapCut（海外）。
- v0.1 仅做**静态滤镜路径**：图像滤镜 + LUT + 简单贴纸；不做交互逻辑、不做面部追踪。
- 复杂面部特效不通过剪映出包（剪映滤镜包能力有限），用户应优先通过抖音 Effect Creator 导出。

### 5.2 节点映射约定

| FX-IR 节点 | 剪映滤镜包表达 |
|---|---|
| `image.lut` | LUT 文件 |
| `image.adjust` | 滤镜参数（亮度 / 对比度等） |
| `image.compose`（仅静态） | 多层贴图叠加 |
| `track.*` / `logic.*` / `model.*`（运行时） | 全部需烘焙为静态结果，否则降级失败、拒绝出包 |

### 5.3 资产规范

- LUT：标准 .cube / .3dl 格式（v0.1 锁 .cube）。
- 贴纸：PNG（带 alpha）。
- 包大小：以平台公开上限为准。

### 5.4 与抖音导出器的关系

- 共享 `core/safety/`、`core/assets/`、烘焙逻辑。
- 平台特定的能力清单与映射表互不影响。

## 6. 安全合规复扫

- 导出器在 export 前调用 `core/safety/asset_scanner.py`：
  - 扫描所有 `Image` / `Sequence<Image>` 资产
  - 过 NSFW classifier
  - 命中即拒绝出包，写审计日志
- 即使节点层已经过滤，导出层也必须再过一遍（双保险）。

## 7. AI 生成元数据标注

- 默认 `ExportOptions.ai_generated_label = True`，不允许关闭（v0.1 锁定）。
- DYFX：写入 `douyinfx_meta.json` 的 `ai_generated: true` 与 `generators: [...]`（含模型 ID）。
- 剪映：写入滤镜包的 `meta.json` 的同等字段（如平台支持）。
- 平台未来要求平台原生标注协议时，按平台规范同步。

## 8. 错误与报告

### 8.1 错误等级

| 等级 | 含义 | 出包行为 |
|---|---|---|
| `error` | 阻塞性问题（资产超限、安全命中、缺关键节点） | 拒绝出包 |
| `warning` | 非阻塞但需要注意（自动降级、参数 clamp） | 出包，报告标注 |
| `info` | 普通信息（耗时、资产数） | 出包 |

### 8.2 HTML 报告结构

- 概览：成功 / 失败、目标平台 / 版本、总耗时
- 资产清单：哈希、原大小、压缩后大小、来源（生成元数据）
- 降级清单：原节点 → 实际表达 / 烘焙
- 警告 / 错误清单：分级 + 节点定位
- 平台规范对照：本工程触及的平台限额（已用 / 上限）

## 9. 平台版本兼容

- 每个 Exporter 维护 `mappings/<platform_version>.py`。
- 新平台 IDE 版本发布后：
  1. 抓 changelog
  2. 跑回归（`tests/exporters/<platform>/regression/`）
  3. 必要时新加 mapping 文件
  4. 老 mapping 至少保留 2 个版本（兼容老用户）

## 10. v0.1 / v1 / v2 路线图

| 平台 | 阶段 | 范围 |

## 11. 不做清单（明确）

- ✗ 不绕过任何平台官方审核 / 上架流程。
- ✗ 不内置任何"刷量"批量导出 API（v0.1 不开放公开 CLI 批处理）。
- ✗ 不修改平台官方编辑器的二进制 / 配置文件（导出器只产出工程目录）。
- ✗ 不复刻平台未公开的私有协议（仅基于平台公开开发者文档与官方授权资源）。
