# 08 - 接口契约

> Project DOUYIN-FX ｜ v0.1
> 文档状态：草案
> 本文档定义跨模块的稳定契约：FX-IR JSON Schema、节点 IO 协议、IPC 协议、Exporter 接口。
> 所有变更必须走版本号 + 迁移器，不允许破坏性修改。

## 1. 契约清单

| 契约 | 作用 | 版本 |
|---|---|---|
| FX-IR JSON Schema | 工程文件格式 | v1 |
| 节点 IO 协议 | 节点端口数据结构 | v1 |
| IPC 协议（Rust ↔ Python） | 子进程通信 | v1 |
| ModelProvider 接口 | 模型实现规范 | v1 |
| Exporter 接口 | 导出器实现规范 | v1 |
| 资产仓协议 | 资产寻址 / 元数据 | v1 |

## 2. FX-IR JSON Schema (v1)

### 2.1 顶层结构

```json
{
  "schema_version": 1,
  "tool_version": "0.1.0",
  "id": "proj_2f1a...",
  "meta": { "name": "...", "author": "...", "created_at": "...", "updated_at": "...", "description": "..." },
  "tracks": [ ... ],
  "nodes": [ ... ],
  "edges": [ ... ],
  "assets": [ ... ],
  "export_targets": { ... },
  "consents": { ... }
}
```

### 2.2 Tracks

跟踪锚点声明（人脸 / 头部 / 手 / 身体 / 场景 / 分割）。

```json
{
  "id": "track_face_0",
  "type": "face",
  "anchor": "face_mesh_v2",
  "max_count": 1
}
```

支持 `type`：`face | head | hand | body | scene | segmentation_mask`

### 2.3 Nodes

```json
{
  "id": "n_8a3f...",
  "type": "model.text2img",
  "node_version": 1,
  "label": "封面贴图生成",
  "params": { ... },
  "position": { "x": 120, "y": 80 },
  "deprecated": false
}
```

- `params` 的 schema 由节点类型自身声明（详见 `04-节点系统设计.md` §5）。
- `node_version` 用于节点级迁移（独立于 schema_version）。

### 2.4 Edges

```json
{
  "id": "e_001",
  "from": { "node_id": "n_a", "port": "out_image" },
  "to":   { "node_id": "n_b", "port": "in_image" }
}
```

### 2.5 Assets

```json
{
  "id": "asset_3c91...",
  "kind": "image",
  "hash": "sha256:3c91...",
  "path": "assets/3c/91/3c91....png",
  "format": "png",
  "size_bytes": 1048576,
  "width": 1024,
  "height": 1024,
  "generation": {
    "source_node_type": "model.text2img",
    "model_id": "sdxl_turbo",
    "model_version": "1.0",
    "params": { "prompt": "...", "seed": 42, "steps": 4, "cfg": 1.0 },
    "created_at": "2026-06-10T19:50:00+08:00"
  },
  "ai_generated": true,
  "license_chain": [ { "spdx": "CreativeML-OpenRAIL-M", "model_id": "sdxl_turbo" } ]
}
```

`kind` 取值：`image | image_sequence | video | mesh | lut | shader_source | other`

### 2.6 Export Targets

```json
{
  "douyin_effect_creator": {
    "enabled": true,
    "target_version": "x.y",
    "overrides": { "max_resolution": [1080, 1920] },
    "bake_overrides": ["n_8a3f..."]
  },
  "jianying_filter_pack": {
    "enabled": true,
    "target_version": "x.y"
  }
}
```

### 2.7 Consents

```json
{
  "ai_generated_label": true,
  "user_uploaded_assets_copyright": [
    { "asset_id": "asset_aa...", "confirmed_at": "..." }
  ],
  "remote_endpoints": [
    { "endpoint_hash": "...", "remember_in_project": true }
  ]
}
```

### 2.8 工程目录布局

```
my_project/
├── project.fxir.json
├── assets/
│   ├── 3c/91/3c91....png
│   ├── 7f/2a/7f2a....mp4
│   └── ...
├── bake/
│   └── n_8a3f.../<frames...>
└── exports/
    ├── douyin_effect_creator/
    │   └── <DYFX 工程目录>
    └── jianying_filter_pack/

```

## 3. 节点 IO 协议（v1）

### 3.1 端口数据结构

```python
class Image:
    data: np.ndarray          # uint8 (H, W, 4) 或 float32 (H, W, 4)
    timestamp_ms: int
    color_space: str          # "srgb" / "linear"
    origin: str               # "top_left" / "bottom_left"

class Mask:
    data: np.ndarray          # float32 (H, W) [0, 1]
    timestamp_ms: int

class Mesh:
    vertices: np.ndarray      # (N, 3) float32
    uv: np.ndarray            # (N, 2) float32
    indices: np.ndarray       # (M, 3) int32
    timestamp_ms: int

class Landmarks:
    points: np.ndarray        # (N, 3) float32
    schema: str               # "facemesh_v2_468" / "mediapipe_hand_21"
    timestamp_ms: int

class Pose:
    matrix: np.ndarray        # (4, 4) float32
    timestamp_ms: int

class Trigger:
    active: bool
    confidence: float
    timestamp_ms: int

class Asset:
    id: str
    kind: str                 # 与 FX-IR Asset.kind 一致
    hash: str
    path: str

class Sequence[T]:
    items: list[T]
    fps: float
```

### 3.2 端口类型 ID 注册

| ID | 类型 |
|---|---|
| `port.image` | `Image` |
| `port.mask` | `Mask` |
| `port.mesh` | `Mesh` |
| `port.landmarks` | `Landmarks` |
| `port.pose` | `Pose` |
| `port.float` / `port.vec2` / `port.vec3` / `port.vec4` | scalar / vec |
| `port.trigger` | `Trigger` |
| `port.asset` | `Asset` |
| `port.sequence.image` | `Sequence[Image]` |

### 3.3 隐式转换

- `Sequence[Image] → Image`：取当前帧（按 `timestamp_ms` 对齐）。
- 其他类型转换需用显式转换节点。

## 4. IPC 协议（Rust ↔ Python，v1）

### 4.1 通道

- 单进程内 stdio JSON-RPC（`Content-Length:` 帧 + JSON body，类 LSP）。
- 重型大对象（图像）走共享内存（mmap），JSON 中只传 handle。

### 4.2 方法集

| 方法 | 方向 | 说明 |
|---|---|---|
| `ping` | R→P | 健康检查 |
| `engine.load_project` | R→P | 加载 FX-IR |
| `engine.save_project` | R→P | 保存 FX-IR |
| `engine.run_preview_frame` | R→P | 跑一帧预览 |
| `engine.bake_node` | R→P | 烘焙某节点 |
| `engine.export` | R→P | 触发导出 |
| `engine.cancel` | R→P | 取消当前任务 |
| `progress.update` | P→R | 任务进度 |
| `safety.event` | P→R | 安全事件通知 UI |
| `log.emit` | P→R | 日志转发 |

### 4.3 错误模型

```json
{ "error": { "code": "E_NODE_EXEC_FAILED", "message": "...", "node_id": "...", "details": { ... } } }
```

错误码列表维护在 `crates/ipc/src/error_codes.rs` 与 `py/core/ipc/error_codes.py`，需保持一致。

## 5. ModelProvider 接口（v1）

详见 `06-模型集成与后端路由.md`。本节给最小契约：

```python
class ModelProvider(Protocol):
```

`payload` / 返回值的 schema 按 capability 单独定义在 `py/models/_schemas/<capability>.py`。

## 7. 资产仓协议（v1）

### 7.1 寻址

- 内容寻址：`hash = sha256(file_bytes)`，前缀分桶 `assets/{hash[:2]}/{hash[2:4]}/{hash}.{ext}`。
- 工程 JSON 仅引用 hash + path（path 与 hash 强一致）。

### 7.2 元数据

每个资产对应 sidecar：`{hash}.meta.json`，结构同 §2.5 `Assets.generation` 字段。

### 7.3 GC

- 工程保存时，扫描 `assets/` 与 FX-IR 引用集合，删除孤儿资产（用户可关闭 GC）。
- 烘焙缓存 `bake/` 按 LRU + 大小上限淘汰。

## 8. 版本与迁移

### 8.1 schema_version

| 版本 | 状态 | 备注 |
|---|---|---|
| v1 | 当前 | v0.1 锁定 |

### 8.2 迁移规则

- **加字段 + 默认值**：不升 schema_version，加载时填默认。
- **改语义 / 删字段 / 改类型**：升 schema_version，在 `py/core/fxir/migrate/v{n}_to_v{n+1}.py` 实现迁移。
- 加载工程时按需链式迁移：v1 → v2 → v3...
- 工具永远兼容历史 schema_version；不支持的最低版本由 changelog 公布并提供脱机迁移工具。

### 8.3 节点版本

- 节点 `node_version` 独立于 schema_version。
- 节点参数变更走"加字段 + 默认值"或迁移函数（`py/nodes/<type>/_migrate.py`）。

## 9. 兼容性测试

- `tests/fxir/regression/`：保存历史版本工程样本，CI 每次跑加载 / 保存 / 等价性校验。
- `tests/exporters/<platform>/regression/`：每个平台维护示例工程 + 期望输出快照。
- 任何破坏兼容性的 PR 必须附迁移器 + 回归用例，否则 CI 拒绝合并。

## 10. 公开 API 与稳定性承诺

- v0.1：所有契约均**草案级**，可能调整；正式承诺稳定性从 v1.0 开始。
- v1.0+：FX-IR schema、节点 IO、Exporter 接口 标注稳定，破坏性变更必须升主版本。
- IPC 协议、ModelProvider 接口 视为内部实现，可在 minor 版本中调整，但需附迁移说明。
